# Sample Data Cleanup Plan

## 목적

현재 `data/samples` 아래에는 데모용 샘플, 메인 계정 실전 샘플, QA 중 생성된 보조 계정 샘플이 함께 섞여 있다.  
이 문서는 어떤 샘플을 기준 데이터로 유지하고, 어떤 샘플을 QA 생성분으로 분류할지 빠르게 판단하기 위한 메모다.

기준 시점: 2026-04-13  
현재 샘플 총량: 약 22.3 MB

## 추천 정리안

### 추천안 A. 안정성 우선

유지:
- 모든 현재 샘플 유지

적합한 경우:
- 라이브 데모와 회귀 QA에서 다양한 계정/챔피언/포지션 커버리지를 그대로 보존하고 싶을 때

장점:
- 가장 안전하다.
- 지금까지 확인한 생성/상세 흐름 결과를 그대로 남길 수 있다.

단점:
- 샘플 수가 계속 늘어난다.

### 추천안 B. 균형형

유지:
- 데모 샘플 2개 전체
- 메인 계정 샘플 전체
- 보조 계정 QA 생성 샘플 중 1개만 유지

정리 후보:
- `sample-kr-8041349600`
- `sample-kr-8048864259`

보존 추천:
- `sample-kr-8048821726`

이유:
- `sample-kr-8048821726`는 최종 수정 후 실제 `generate-sample 1회 -> sample load -> DETAIL_VIEW` 검증에 직접 사용된 샘플이다.
- 나머지 2개는 참고 가치가 있지만, 현재 핵심 회귀 시나리오 재현에는 필수는 아니다.

### 추천안 C. 최소 데이터형

유지:
- `sample-001`
- `sample-002`
- 메인 계정 샘플 전체
- `sample-kr-8065601119`
- `sample-kr-8097520888`

정리 후보:
- `sample-kr-8041349600`
- `sample-kr-8048821726`
- `sample-kr-8048864259`

적합한 경우:
- 이번 QA 생성분은 모두 일회성 산출물로 보고, 기존 기준 데이터만 유지하고 싶을 때

주의:
- 최종 생성 경로를 통과한 최신 샘플 흔적이 사라진다.
