## 1. 문서 목적

이 문서는 MVP용 `샘플 경기 1건`을 어떤 기준으로 선택할지 정의한다.

목표는 아래와 같다.

- 분석 품질이 잘 드러나는 경기를 고른다
- UI 데모에 적합한 서사가 있는 경기를 고른다
- 장점/단점/체크리스트가 자연스럽게 나오는 경기를 고른다
- 실제 Riot API 수집 후 후보 경기들 중 빠르게 채택 여부를 판단할 수 있게 한다

## 2. 핵심 원칙

- 샘플 경기는 `통계만 많은 경기`보다 `해석 가능한 흐름이 있는 경기`를 우선한다
- 너무 일방적인 stomp 게임보다 `장점과 단점이 함께 드러나는 경기`를 우선한다
- 사용자가 보자마자 이해할 수 있는 챔피언/포지션 조합을 우선한다
- 샘플 경기는 `분석 데모`를 위한 것이므로, 경기 자체의 수준보다 `분석 가능성`이 중요하다

## 3. 샘플 경기의 역할

첫 샘플 경기는 아래 역할을 동시에 수행해야 한다.

- 랜딩 페이지 데모용 결과 화면
- 분석 JSON 스키마 검증용 입력 데이터
- 프론트엔드 컴포넌트 연결 테스트용 데이터
- LLM 프롬프트 품질 평가용 기준 데이터

즉, 첫 샘플은 단순 예시가 아니라 `기준 경기`다.

## 4. 필수 조건

아래 조건을 모두 만족해야 샘플 후보로 인정한다.

### 경기 유형

- `Summoner's Rift`
- `랭크 솔로` 또는 `일반 게임`
- `custom match` 제외

### 경기 길이

- 권장 범위: `28~32분`
- 허용 범위: `25~35분`

이유:

- 너무 짧으면 중반/후반 분석이 약해진다
- 너무 길면 MVP 데모에 필요한 핵심 포인트가 흐려질 수 있다

### 역할 명확성

- 플레이어 역할이 분명해야 한다
- `TOP`, `JUNGLE`, `MID`, `ADC`, `SUPPORT` 중 하나로 자연스럽게 읽혀야 한다

### 주요 서사 존재

아래 4개 중 최소 3개는 있어야 한다.

- 초반 잘한 장면
- 중반 흐름 전환 장면
- 오브젝트 관련 장면
- 교전/합류 판단 장면

### 분석 출력 가능성

아래 결과가 자연스럽게 생성 가능해야 한다.

- 장점 3개
- 단점 3개
- 행동 체크리스트 3개 이상
- 핵심 장면 4개 이상

## 5. 우선 추천 샘플 프로필

첫 샘플은 아래 프로필을 가장 우선 추천한다.

- 역할: `MID`
- 결과: `LOSS`
- 경기 길이: `29~31분`
- 챔피언: 대중적이고 이해하기 쉬운 챔피언
- 흐름: `초반은 좋았지만 중반 실수로 흐름을 잃은 경기`

이 프로필이 좋은 이유:

- 장점과 단점이 균형 있게 나온다
- 코칭형 체크리스트가 잘 나온다
- 경기 요약 문장이 자연스럽게 만들어진다
- 일반 사용자도 장면을 이해하기 쉽다

## 6. 추천 챔피언 유형

첫 샘플에서는 아래 타입의 챔피언을 우선 고려한다.

### 1순위

- Ahri
- Orianna
- Syndra
- Taliyah

이유:

- 라인전, 로밍, 교전 합류, 포지셔닝 설명이 쉽다
- 너무 복잡한 메커니즘 없이도 경기 해석이 자연스럽다

### 2순위

- Lee Sin
- Jarvan IV
- Jinx
- Thresh

이유:

- 분석 포인트는 풍부하지만, MVP 첫 샘플로는 해설 난이도가 조금 높을 수 있다

### 후순위

- 지나치게 특수한 운영 챔피언
- 해석이 챔피언 전문지식에 과도하게 의존하는 챔피언
- 사용자 이해가 어려운 비주류 조합

## 7. 피해야 할 경기 유형

아래 조건 중 하나라도 강하게 해당하면 샘플 채택을 피한다.

### 너무 짧은 경기

- 20분 이하

문제:

- 초반/중반/후반 구분이 흐려진다
- 체크리스트가 단조로워진다

### 너무 일방적인 stomp

- 초반부터 끝까지 한쪽이 압도

문제:

- 장점/단점이 한쪽으로 치우친다
- 코칭형 피드백이 뻔해진다

### 역할 불명확 경기

- 포지션이 애매하거나 비정형

문제:

- 분석 결과가 일반론으로 흐를 가능성이 높다

### 이벤트가 부족한 경기

- 오브젝트/교전/로밍 서사가 거의 없음

문제:

- 타임라인과 근거 이벤트가 빈약해진다

### 정책상 애매한 경기

- custom queue
- 공개 처리 위험이 있는 데이터

## 8. 샘플 평가 항목

후보 경기를 아래 항목으로 평가한다.

## 항목 A. 서사성

질문:

- 경기 흐름을 한 문장으로 설명할 수 있는가
- 초반/중반/후반에 서로 다른 해석 포인트가 있는가

점수:

- 1점: 거의 서사가 없음
- 3점: 기본 서사는 있음
- 5점: 뚜렷한 전환점과 코칭 포인트가 있음

## 항목 B. 장점 도출 가능성

질문:

- 장점 3개를 억지 없이 뽑을 수 있는가

점수:

- 1점: 장점이 거의 없음
- 3점: 장점 2~3개 가능
- 5점: 명확한 장점 3개 이상 가능

## 항목 C. 단점 도출 가능성

질문:

- 단점 3개를 근거 있게 뽑을 수 있는가

점수:

- 1점: 단점이 모호함
- 3점: 단점 2~3개 가능
- 5점: 명확한 단점 3개 이상 가능

## 항목 D. 체크리스트 전환 가능성

질문:

- 다음 게임 행동으로 바로 번역 가능한가

점수:

- 1점: 행동 아이템으로 바꾸기 어려움
- 3점: 일부 가능
- 5점: 3개 이상 자연스럽게 가능

## 항목 E. UI 데모 적합성

질문:

- 결과 페이지 카드와 타임라인이 풍성하게 채워지는가

점수:

- 1점: UI가 빈약하게 보임
- 3점: 기본은 채워짐
- 5점: 결과 페이지가 설득력 있게 채워짐

## 항목 F. 일반 사용자 이해도

질문:

- 사용자가 챔피언/포지션/장면을 쉽게 이해할 수 있는가

점수:

- 1점: 이해 난이도가 높음
- 3점: 보통
- 5점: 누구나 보기 쉬움

## 9. 채택 기준 점수표

총점 30점 만점으로 평가한다.

- 25~30점: 즉시 채택 가능
- 21~24점: 보조 후보
- 20점 이하: 샘플로 비추천

추가 규칙:

- `서사성` 4점 미만이면 탈락
- `체크리스트 전환 가능성` 4점 미만이면 탈락
- `UI 데모 적합성` 4점 미만이면 탈락

## 10. 실제 선정 절차

### Step 1. 후보 5경기 확보

- 최근 경기 5개를 우선 가져온다

### Step 2. 1차 필터

아래 조건으로 제거한다.

- 25분 미만
- custom match
- 역할 불명확
- 서사 부족

### Step 3. 2차 점수화

- 남은 경기들을 30점 기준으로 평가한다

### Step 4. 상위 2경기 비교

비교 기준:

- 더 설명하기 쉬운가
- 더 균형 잡힌 장점/단점이 나오는가
- 더 타임라인이 풍성한가

### Step 5. 최종 1경기 채택

- 원본 JSON 저장
- 정규화 데이터 생성
- 분석 결과 JSON 생성

## 11. 첫 샘플의 권장 테마

첫 샘플은 아래 테마를 권장한다.

`초반 주도권과 로밍은 좋았지만, 중반 시야/오브젝트 준비 부족과 무리한 합류 타이밍으로 흐름을 잃은 경기`

이 테마는 아래 장점이 있다.

- 요약 문장이 자연스럽다
- 장점과 단점을 동시에 보여준다
- 체크리스트 품질이 높다
- 미드/정글/원딜 등 여러 역할에도 재사용 가능한 분석 패턴이다

## 12. 백업 샘플 전략

첫 샘플 외에 바로 이어서 준비할 백업 샘플도 계획한다.

### 백업 샘플 A

- `정글`
- `승리 경기`
- 초반 주도권을 오브젝트로 잘 연결한 경기

목적:

- 첫 샘플이 패배 경기 중심일 때 균형 보완

### 백업 샘플 B

- `원딜 또는 서포터`
- 후반 한타/포지셔닝 평가가 잘 드러나는 경기

목적:

- 역할별 분석 확장성 검증

## 13. 익명화 원칙

- 공개 샘플에서는 Riot ID를 가명 처리한다
- 내부 원본 데이터와 공개용 샘플 데이터를 분리한다
- 프론트엔드에 직접 노출되는 이름은 익명화 버전을 사용한다

## 14. 현재 권장 결정

현재 시점의 첫 샘플 경기 선정 기준은 아래처럼 확정한다.

- `랭크 솔로 또는 일반 게임`
- `MID`
- `LOSS`
- `29~31분`
- `대중적 챔피언`
- `초반 장점 + 중반 실수 + 오브젝트 전환점 + 후반 종결 구도`가 모두 존재
